## 理论知识

### 1. 什么是知识切分

知识切分，就是把原始文档拆成更适合检索的小块，再分别做 embedding 和入库。

例如一篇长文档里可能同时包含：

- Token
- Context Window
- Function Calling
- Agent Loop

如果整篇作为一个向量入库，用户只问 “什么是 Agent Loop”，检索时可能把整篇都召回，信息太杂，相关性反而下降。

### 2. 为什么不能总是整篇入库

整篇入库的问题：

- 一个 chunk 主题太多，向量语义被稀释
- 检索到的内容太长，Prompt 噪音变大
- 明明命中了，但真正有用的信息只占一小部分

### 3. chunk 太小的问题

如果切得太碎，也会有问题：

- 语义上下文不完整
- 回答问题时缺少前后信息
- 检索命中后仍然看不懂完整含义

### 4. chunk 太大的问题

如果 chunk 太大：

- 多个主题混在一起
- 查询和 chunk 的相似度不够聚焦
- 召回质量下降

### 5. RAG 检索质量的几个关键因素

一个 RAG 系统的效果，通常至少取决于：

1. 知识切分质量
2. embedding 模型质量
3. 检索 top_k 设置
4. 是否需要 metadata 过滤
5. Prompt 如何注入检索结果

今天重点先抓第 1 个：知识切分。
