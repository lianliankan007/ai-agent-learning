# agent loop的核心逻辑
- 1. 构建初始消息列表，包含系统提示和历史对话
- 2. 循环执行，直到达到最大步数或模型不再调用工具
- 3. 每一步调用模型，解析工具调用指令，执行工具，并把结果反馈给模型
- 4. 最后输出模型给出的最终答案，或者在达到最大步数后停止并给出提示

# agent loop 框架如下：
```
    messages = [system_prompt] + history + [user_message]
    for step in range(max_steps):
        assistant_message = call_model(messages, tools)
        messages.append(assistant_message)
        if assistant_message doesn't call tools:
            return assistant_message.content
        for each tool_call in assistant_message:
            tool_result = execute_tool(tool_call)
            messages.append(tool_result)    
    return "达到最大步数，停止执行"
```
