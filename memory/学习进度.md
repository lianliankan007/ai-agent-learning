## day19 学习内容
```
- 工具失败不是异常分支，而是 Agent 工程中的常态
- retry：适合处理限流、超时、网络抖动等临时错误
- fallback：主工具失败后切换到缓存或替代工具
- 程序层要控制重试次数、错误分类和降级策略
- 最终回答要说明结果边界，不能把降级结果伪装成实时结果
```
