## 理论知识

### 1. 为什么会出现多轮 Function Calling

有些问题不是一次工具调用就能完成。

例如：

- 先查长期记忆，再结合知识库回答
- 先查资料，再做总结
- 先计算，再根据结果给建议

这种情况下，模型在拿到第一次 observation 后，可能还需要继续决定：

- 还要不要再调工具
- 下一个工具是什么
- 还是现在可以直接给最终答案

### 2. 什么是最小 Agent Loop

最小 Agent Loop 可以理解成：

1. 用户提问
2. 模型决定下一步动作
3. 如果要调工具，就执行工具
4. 把 observation 返回给模型
5. 模型继续决定
6. 直到输出最终答案

这个循环，就是 Agent 和普通单次问答最大的区别之一。

### 3. 为什么需要 `max_steps`

如果没有步数限制，Agent 可能会：

- 一直重复调用工具
- 在错误状态里打转
- 进入死循环

所以真实 Agent 通常会设置：

- `max_steps`
- 明确退出条件

这样系统才能可控。

### 4. 什么是退出条件

退出条件常见包括：

1. 模型明确输出最终答案
2. 已达到 `max_steps`
3. 工具调用失败，无法继续
4. observation 已足够支撑回答

### 5. Day18 为什么重要

如果 Day17 学的是：

- 单次结构化工具调用

那么 Day18 学的是：

- 工具调用如何进入一个最小的智能体循环

这一步非常关键，因为后面真正的 Agent 架构，都会建立在这种循环能力之上。
