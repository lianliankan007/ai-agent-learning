## 理论知识

### 1. 一个最小 RAG 系统由哪些环节组成

一个最小但真实的 RAG 系统，通常至少包含：

1. 原始知识
2. chunk 切分
3. embedding 向量化
4. 向量库入库
5. 检索策略
6. Prompt 注入
7. 大模型生成答案

这 7 步中，任何一步出问题，最终回答都可能变差。

### 2. RAG 答得不好，不一定是模型的问题

很多初学者第一反应是：

- 模型太弱
- Prompt 不够好

但真实情况里，更常见的问题其实是：

- chunk 不合理
- embedding 表达不准
- top_k 太大或太小
- 没有加过滤条件
- 检索到了错误知识
- 把太多噪音一起塞进 Prompt

### 3. 如何区分“检索问题”和“生成问题”

可以这样判断：

1. 如果检索结果本身就不对
   - 这是检索问题
2. 如果检索结果是对的，但最终回答没用上
   - 更像 Prompt 或生成问题
3. 如果检索结果部分正确，但混入很多噪音
   - 更像检索策略问题

### 4. Day14 为什么重要

如果没有“问题定位能力”，后面你做更复杂的 Agent 项目时会出现一个常见问题：

- 效果不好，但不知道改哪里

所以 Day14 本质上是在训练你形成一个工程思维：

- 不只是看到“结果不好”
- 而是能拆成链路，逐段判断哪里出了问题
