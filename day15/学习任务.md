## 理论知识

### 1. 什么是 Tool Use

Tool Use 的本质是：

- 模型先判断当前问题是否需要借助外部能力
- 如果需要，就输出调用哪个工具、传什么参数
- 程序执行工具
- 再把工具结果返回给模型
- 最后由模型基于工具结果生成答案

所以 Tool Use 本质上是在扩展模型能力边界。

### 2. 为什么不能只靠模型直接回答

如果完全不调用工具，会出现几个问题：

- 遇到计算题时，纯语言推理可能不稳定
- 遇到外部数据时，模型看不到实时信息
- 遇到项目知识时，模型可能没有相关上下文
- 遇到需要执行动作时，模型只能“说”，不能“做”

所以工具调用常见用途包括：

- 计算
- 检索知识
- 查询外部系统
- 调用 API
- 操作本地程序

### 3. 什么情况下应该调用工具

通常至少包括这些场景：

1. 需要精确计算
2. 需要外部知识或数据库
3. 需要实时信息
4. 需要执行动作
5. 需要访问模型参数里没有的信息

### 4. 什么情况下不应该调用工具

也有很多情况其实不需要工具：

- 简单概念解释
- 常规总结
- 一般性表达优化
- 不依赖外部信息的开放式回答

如果本来可以直接回答，却强行调用工具，系统会变慢，也会增加复杂度。

### 5. 最小 Tool Use 链路

一个最小 Tool Use 流程通常是：

1. 用户提问
2. 判断是否需要工具
3. 选择合适工具
4. 组装参数
5. 执行工具
6. 获取 observation
7. 返回最终答案

Day15 的目标，就是先把这条链路理解清楚。
