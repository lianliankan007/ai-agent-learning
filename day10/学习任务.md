## 理论知识

### 1. Day9 到 Day10 的关键变化

Day9 你做的是“语义检索模拟”，本质上仍是教学型向量表示。  
Day10 的核心变化是：

- Day9：手工特征 + 规则映射（教学）
- Day10：真实 Embedding 向量（工程）

你可以把 Day10 看成“从概念正确，走向实现正确”。

### 2. Embedding 在 RAG 中到底负责什么

Embedding 负责把文本映射到向量空间，让“语义接近”的文本在空间里更靠近。

RAG 检索层通常流程是：

1. 文档切分
2. 文本转向量（Embedding）
3. 向量入库（向量数据库或本地索引）
4. 查询时把问题也 Embedding
5. 做相似度检索（Top-K）
6. 返回候选上下文给生成层

### 3. 为什么 Top-K 很重要

Top-K 不是越大越好。太小会漏信息，太大会引入噪声。  
常见做法是：

- 先用较小 K（如 3 到 5）做 baseline
- 根据问答质量和 token 成本再调整

### 4. Embedding 检索的三个常见误区

1. 只看“有没有命中”，不看排序质量
2. 不做文本切分，直接整篇文档向量化
3. 不观察检索分数分布，盲目相信第一名
