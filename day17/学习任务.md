## 理论知识

### 1. 什么是 Function Calling

Function Calling 的核心思想是：

- 模型不是直接返回普通自然语言
- 而是按约定格式返回：
  - 要不要调用工具
  - 调哪个工具
  - 参数是什么

程序拿到这个结构化结果后，再真正执行工具。

### 2. 为什么要结构化

如果模型只是输出一句自然语言：

- “我觉得应该调用计算器，参数可能是 18 * 6 + 3”

程序还得自己再猜一遍，稳定性很差。

如果模型按结构化格式输出：

- `tool_name=calculator`
- `arguments={"expression":"18 * 6 + 3"}`

程序就能更稳定地解析和执行。

### 3. 什么是函数描述

函数描述就是工具的“使用说明书”，通常至少包含：

1. 工具名
2. 工具作用
3. 参数名
4. 参数类型
5. 参数语义

函数描述越清晰，模型越容易正确调用工具。

### 4. Day17 为什么重要

真正的 Agent 系统里，Tool Use 往往不是纯规则硬编码，而是：

- 模型先根据工具描述做决策
- 再输出结构化调用请求

所以 Day17 本质上是在帮助你理解：

- Function Calling 是 Tool Use 的标准化表达方式
